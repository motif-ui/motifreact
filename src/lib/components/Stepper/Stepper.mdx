### Controlled vs Uncontrolled

Stepper component can be used in both controlled and uncontrolled modes. By default, it operates in uncontrolled mode,
where it manages its own state internally. However, if you want to have more control over the stepper's behavior, you
can use it in controlled mode by using the `useStepper` hook. This allows you to manage the state of the stepper and
have more control over its behavior.

### useStepper

useStepper hook is designed to extend the functionality of Stepper component. It provides necessary props/functions for:

- Data Storing
- Custom Navigation

> **Note:** The return of useStepper should be passed to Stepper's `state` prop to work properly.

<br />

#### Api

```tsx
type UseStepperReturn = {
  activeStep: number;
  stepData: Partial<Record<number, Record<string, unknown>>>;
  setStepData: (index: number, data: Record<string, unknown>) => void;
  goToStep: (index: number) => void;
  goToNextStep: () => void;
  goToPrevStep: () => void;
};
```

#### Data Storing

`useStepper` provides a `data` object and a `setData` function to store and update data across different steps. This
takes over the burden from the user to manage the state of the data across steps, allows you to maintain state and pass
information between steps easily.

#### Custom Navigation

Stepper component handles the navigation between steps out of the box. However, if you want to control the navigation
manually, you can use `useStepper` hook. It provides the current step index and functions to change the step index,
allowing you to control the navigation between steps manually.

It is possible to freely jump between steps using the provided function, or to implement custom navigation logic based
you own conditions.
