# Icons

Motif UI does not include an embedded icon library by default. Instead, it provides a flexible way to use icons in your
project. You can choose to use our default set, your own icon set, other icon libraries around or any other SVG icon
set. Those included icons can be used in `<Icon>`, `<IconButton>` or the `icon` prop some components.

It is always welcome and supported to use methods like svgs, etc. but Font Icons with ligature support is the preferred
and the easiest way to apply and use the icons, library-wide.

<br />

## Motif UI Icons

In fact, we have our own set of font icons. If you think those icons are enough for your app, the easiest way to use
them is to just import the necessary css file in your app's root layout. All other things are already set and there is
nothing to do more!

```jsx
// ...your root layout file
<link rel="stylesheet" href="https://cdn.jsdelivr.net/gh/motif-ui/assets/css/motif-icons.css" />

// ...you component
<Icon name="folder" />
```

The name of the icons available can be found in `<Icon>`, `<IconButton>` component pages.

<br />

## Other Libraries

There are several options around but the most popular ones are **FontAwesome** and **Material Icons**. You are always
welcome to use any other icon font library as well.

In order to use a custom font icon library, you should include the desired icon library in your project and then set the
`baseIconClassName` prop in **MotifProvider** with the base class name of the icon library you are using.

```jsx
// ...your root layout file
<link rel="stylesheet" href="https://fonts.googleapis.com/icon?family=Material+Icons" />

<MotifProvider baseIconClass="material-icons">
  {/* your app's children */ }
</MotifProvider>
```

Then, in anywhere in your app, `icon` props of the components or `<Icon>`, `<IconButton>` components can use the icon
names defined by the icon library.

```jsx
import { Icon, Avatar, IconButton } from "@motif-ui/react";

<Icon>android</Icon>
<IconButton name="android" />
<Avatar icon="android" />
```
